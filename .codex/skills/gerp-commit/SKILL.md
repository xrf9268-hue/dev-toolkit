## 任务

创建一个规范化的 git 提交。

## 执行步骤

1. 获取当前分支名
2. 确定 JIRA 前缀（优先级：用户参数 > 分支提取 > 无前缀）
3. 查看工作区和暂存区变更（无变更则停止）
4. 分析变更内容并选择性暂存
5. 提议 commit message，并展示暂存区 diff/stat
6. 用户确认后创建提交

## JIRA 前缀规则

1. 如果用户指定了 JIRA 编号（如 `BGERP-12345`），使用用户指定的
2. 否则从分支名中提取（匹配 `BGERP-[0-9]+` 模式）
3. 如果都没有，允许无前缀提交

**分支示例：**
- `yvan/BGERP-32921-售后看板` → 提取 `BGERP-32921`
- `feature/BGERP-12345-xxx` → 提取 `BGERP-12345`
- `main` / `master` → 无前缀

## 提交格式规范

### 格式要求

1. **JIRA 前缀**：使用中文方括号包裹，如 `【BGERP-32921】`
2. **语言**：使用中文描述变更内容
3. **首行**：简短描述做了什么（50字符以内）
4. **正文**（可选）：详细说明原因、影响或技术细节

### 禁止内容

- 不要使用 emoji
- 不要使用英文 commit type 前缀（feat:, fix:, chore: 等）
- 不要在未确认暂存内容时直接提交

### 示例

**有 JIRA 时：**
```
【BGERP-32921】修复分页下拉菜单被水平滚动条遮挡问题

根因：表格容器设置了 z-index 创建了局部层叠上下文
修复：删除父级容器的 z-index 属性
```

**无 JIRA 时：**
```
修复分页下拉菜单被水平滚动条遮挡问题
```
